Duration: 60 minutes (Part 1 of 3) Format: Asynchronous self-paced learning

Learning Objectives

By the end of this module, you will be able to:

  • Distinguish between hardware and software problems
  • Use a systematic diagnostic process
  • Recognize common hardware failure patterns
  • Guide users through diagnostic steps remotely
  • Apply diagnostic thinking to realistic scenarios

Section 2.1: Hardware vs. Software - Making the Distinction

Time: 10 minutes

Why This Matters

As a consultant, one of your most important skills is quickly determining: "Is this a hardware problem or a software problem?" This determines:

  • Whether you can help remotely or need to escalate
  • Whether the issue can be fixed or requires replacement
  • How urgent the situation is

The Challenge

Many symptoms can be caused by EITHER hardware OR software:

  • Slowness
  • Crashes
  • Display problems
  • File corruption
  • Startup issues

So how do you tell the difference?

Key Distinguishing Questions

1. When does it happen?

Hardware issues:

  • Occur at specific times (when hot, when cold, when moved)
  • Happen before OS fully loads (during BIOS/startup)
  • Correlate with environmental conditions
  • Progressive (get worse over time)

Software issues:

  • Happen with specific programs or tasks
  • Occur after OS has loaded
  • May start after software installation/update
  • May be intermittent but not environmentally linked

Example:

  • Computer crashes when running specific software = likely software
  • Computer crashes more often on hot afternoons = likely hardware (cooling)

2. Is it consistent or variable?

Hardware issues:

  • Usually consistent once triggered
  • Follow patterns (always when hot, always after 20 minutes, etc.)
  • Physical symptoms (beeps, clicking sounds, smell, heat)

Software issues:

  • May be inconsistent or random
  • Sometimes fixed by restart
  • No physical symptoms

3. Does it respond to software fixes?

Hardware issues:

  • Don't improve with software troubleshooting
  • Reinstalling programs doesn't help
  • System restore doesn't fix it
  • Safe Mode doesn't change behavior

Software issues:

  • May improve with updates, reinstalls, or restarts
  • Safe Mode often behaves differently
  • Specific to certain programs or functions

Activity 2.1: Hardware or Software?

Time: 5 minutes

For each scenario, determine if it's more likely hardware or software, and write down your reasoning:

A. Computer freezes only when using video conferencing software. Other programs work fine. Started after recent Windows update.

B. Computer makes three beeps when you press power button, screen stays black. Happened suddenly after transporting laptop in backpack.

C. Files keep getting corrupted. Multiple programs crash. Computer sometimes makes clicking sounds. Gets worse over time.

D. Specific application won't launch - gives error message. All other programs work normally. Computer otherwise functions well.

E. Computer shuts down without warning, especially when running multiple programs. Bottom of laptop feels very hot. Fan is very loud. Been getting worse over 3 months.

Answers

A. Software - Specific to one program, started after update, other programs fine

B. Hardware - Beep codes indicate hardware problem, occurred after physical shock (likely unseated RAM)

C. Hardware - Physical sounds (clicking = failing hard drive), progressive deterioration, affects multiple areas

D. Software - Single application issue, other functions normal, error message (software reporting problem)

E. Hardware - Thermal symptoms (hot, loud fan, shutdowns), progressive, correlates with system load (heat generation)

Key Principle: When in doubt, look for physical symptoms, environmental correlations, and whether the problem crosses multiple programs/functions. Hardware problems usually affect the whole system; software problems are often more specific.

Section 2.2: The Systematic Diagnostic Process

Time: 15 minutes

The Four-Step Framework

Good consultants don't guess - they follow a systematic process. Here's the framework you'll use for every hardware issue:

  1. OBSERVE → 2. ISOLATE → 3. TEST → 4. DECIDE

Let's break down each step:

STEP 1: OBSERVE (Gather Information)

What you're doing: Collecting detailed information about the problem

Key questions to ask:

About the symptom:

  • What exactly happens? (Be specific - "slow" means different things)
  • When did it start?
  • Does it happen every time or intermittently?
  • What were you doing when it happened?

About patterns:

  • When does it happen? (time of day, specific activities, after certain duration)
  • When does it NOT happen?
  • Is it getting worse, better, or staying the same?

About context:

  • Any recent changes? (new software, updates, physical incidents)
  • Any environmental factors? (heat, storm, transport, etc.)
  • What's the work environment like?

About the computer:

  • How old is it?
  • Any previous problems?
  • When was it last cleaned/maintained?

Physical symptoms:

  • Any sounds? (beeps, clicking, grinding, fan noise)
  • Any smells? (burning, electrical)
  • Any heat? (where specifically)
  • Any visible damage?

Why this matters: Good observation often points directly to the cause. "Slow computer" could be anything, but "slow startup, normal after, clicking sound from left side, started after power outage" strongly suggests failing hard drive damaged by power surge.

STEP 2: ISOLATE (Narrow Down Possibilities)

What you're doing: Determining which component or system is involved

Strategies:

Use the symptoms to eliminate possibilities:

  • No display but computer powers on → NOT power system, likely display or graphics
  • Slow but no crashes → NOT likely RAM, probably storage or cooling
  • Happens only on battery → battery or charging system

Use timing clues:

  • During startup → storage or RAM
  • After 20 minutes of use → cooling system
  • Immediately after transport → unseated component
  • Gradually worsening → component degradation

Use environmental clues:

  • More problems in heat → cooling or battery
  • More problems in morning (cold/humid) → condensation, connections
  • After power outage → power system, possibly storage corruption

Consider interactions:

  • Multiple symptoms → might be one failing component affecting others
  • Example: Failing cooling → CPU overheats → throttles → appears slow → crashes

STEP 3: TEST (Confirm Your Hypothesis)

What you're doing: Designing tests to confirm or rule out your hypothesis

Types of tests:

Observation tests (no intervention):

  • Monitor temperatures during use
  • Listen for specific sounds
  • Note when problems occur vs. don't occur
  • Watch startup sequence carefully

Software tests (information gathering):

  • Run built-in diagnostics (Battery Report, CHKDSK, Memory Diagnostic)
  • Check system temperatures with monitoring software
  • Review event logs for error patterns
  • Test with external monitor (isolates display vs. graphics)

Simple hardware tests (user can perform):

  • Disable touchpad, use external mouse (isolates touchpad)
  • Try different power outlet (isolates power supply issues)
  • Boot in Safe Mode (software vs. hardware)
  • Remove peripherals (isolates external device issues)

Guided hardware tests (you walk user through):

  • Reseat RAM
  • Clean vents
  • Check cable connections
  • Let system cool and retry

The key: Choose tests that will clearly distinguish between your top hypotheses. "If it's X, this test will show Y. If it's Z, this test will show W."

STEP 4: DECIDE (Recommend Action)

What you're doing: Determining the appropriate next step based on findings

Decision tree:

IF problem is clearly diagnosable and fixable:

  • Guide user through fix (cleaning, reseating, settings change)
  • Provide preventive advice

IF problem requires part replacement:

  • Assess: Can user continue working? How urgent?
  • Recommend: Specific part needed, where to source it
  • Consider: Workarounds while waiting for part

IF problem is complex or diagnosis uncertain:

  • Escalate to IT support with documentation
  • Explain what you've ruled out (saves them time)

IF problem indicates imminent failure:

  • Immediate data backup
  • Minimize use
  • Request replacement

IF repair isn't practical:

  • Cost/benefit analysis
  • Replacement recommendation
  • Short-term alternatives

Context matters in decisions:

  • Remote location with no parts = different decision than city with tech support nearby
  • Critical deadline next week = different urgency than routine maintenance
  • Expensive repair on old computer = maybe better to replace

Miguel's Scenario (recap):

"My computer is running really slowly now. It used to start up in less than a minute, but now it takes 5-7 minutes. Once it's running, programs work mostly okay, but they take a while to open."

STEP 1 - OBSERVE: What additional questions would you ask Miguel? Write at least 4 specific questions.

STEP 2 - ISOLATE: Based on "slow startup, normal operation after," which component is most likely involved? What are you ruling out?

STEP 3 - TEST: What tests would you have Miguel perform to confirm your hypothesis? List at least 2 tests.

STEP 4 - DECIDE: If your hypothesis is correct, what would you recommend? Consider both immediate action and prevention.

Sample Answers:

OBSERVE questions:

  • Does the computer have an HDD or SSD? (HDDs more likely to fail this way)
  • Do you hear any clicking or grinding sounds during startup?
  • How full is your hard drive?
  • Have you seen any error messages about disk problems?
  • Does the slowness happen equally every time, or vary?

ISOLATE:

Most likely: Storage drive (HDD) failing or severely full

Why: Slow startup suggests loading OS from storage is the bottleneck. Normal operation after suggests CPU and RAM are fine - if they were the problem, slowness would continue during use.

  • Check drive health: Run CHKDSK or manufacturer diagnostic tool
  • Check free space: How full is the drive?
  • Listen carefully during startup: Clicking sound confirms failing drive
  • Check event logs: Look for disk errors

If drive is failing:

  • IMMEDIATE: Back up all data NOW (drive could fail completely)
  • SHORT TERM: Prepare for drive replacement (or computer replacement if old)
  • PREVENTION: Get surge protector immediately (protect new drive)

If drive is just very full:

  • IMMEDIATE: Free up space, move files to external storage
  • ONGOING: Better data management, regular cleanup
  • PREVENTION: Still get surge protector

Reflection: Notice how the systematic process turns "computer is slow" (vague) into "failing hard drive, needs backup and replacement, requires surge protection" (specific, actionable).

Section 2.3: Common Failure Patterns

Time: 20 minutes

Why Patterns Matter

You'll see certain problems again and again. Recognizing these patterns helps you diagnose faster and more accurately.

Pattern 1: Battery Issues

Common symptoms:

  • Won't charge or charges very slowly
  • "Plugged in, not charging" message
  • Battery drains very quickly
  • Computer won't run on battery at all
  • Battery percentage jumps around

Likely causes:

  • Battery degradation (age + heat)
  • Power adapter failure
  • Charging port damage
  • Charging circuitry problem

How to diagnose:

  • Check battery health report (built-in OS tool)
  • Try wiggling power cable (loose port?)
  • Check adapter temperature (should be warm, not extremely hot)
  • Try different outlet (eliminates power source issue)
  • If possible, try different adapter (isolates adapter vs. battery/port)

Common context:

  • Batteries 2+ years old in hot climates
  • Power voltage fluctuations damaging adapters
  • Physical damage to ports from frequent plugging/unplugging

Solutions:

  • Replace battery (if that's the issue)
  • Replace adapter (if that's the issue)
  • Port repair may need professional help
  • Can often continue working plugged in even with dead battery

Prevention:

  • Surge protector/voltage regulator
  • Avoid extreme heat when possible
  • Don't leave plugged in 24/7 in very hot environments
  • Careful with charging port (don't stress connector)

Pattern 2: Overheating and Thermal Shutdown

Common symptoms:

  • Unexpected shutdowns (no warning, just powers off)
  • Shutdowns more common with intensive tasks
  • Very loud fan noise
  • Computer hot to touch (especially bottom/sides)
  • Slowness that improves after cooling
  • More problems during hot parts of day

Likely causes:

  • Dust blocking cooling vents
  • Fan failure
  • Dried thermal paste
  • Working in hot environment

How to diagnose:

  • Check temperature with monitoring software
  • Listen to fan (not running? too loud?)
  • Visual inspection of vents (dust visible?)
  • Note correlation with workload and time of day
  • Check if problem improves after computer cools down

Common context:

  • Dusty environments (dry season especially)
  • Months without cleaning
  • Hot workspaces without AC
  • Working on soft surfaces that block vents

Solutions:

  • Clean vents and fan (compressed air or careful brushing)
  • Improve workspace ventilation
  • Laptop cooling pad if available
  • Fan replacement if fan has failed
  • Work during cooler hours if possible

Prevention:

  • Monthly cleaning in dusty environments
  • Always use on hard, flat surface
  • Keep vents clear
  • Work in coolest available location

Pattern 3: Storage Drive Problems

Common symptoms:

For failing drives:

  • Clicking or grinding sounds
  • Very slow performance (especially startup and file access)
  • Files corrupting or disappearing
  • Frequent crashes
  • "Disk error" messages
  • Computer freezes when accessing files

For full drives:

  • Slow performance
  • "Disk full" warnings
  • Can't save files
  • Programs won't update or install

Likely causes:

  • HDD mechanical failure (clicking = very bad sign)
  • SSD wear (less common but possible)
  • Full drive (no space left)
  • Corruption from power issues
  • Physical shock damage

How to diagnose:

  • Listen for clicking (classic failing HDD sound)
  • Check drive space (how full?)
  • Run diagnostic tools (CHKDSK, manufacturer diagnostics)
  • Note if problems correlate with file access
  • Check SMART status (drive health indicator)

Common context:

  • HDDs in hot/dusty environments or after physical shock
  • Language workers collecting large audio/video files
  • Power outages causing corruption
  • Older drives (3+ years)

Solutions:

  • If failing: Backup immediately, replace drive
  • If full: Free space, move files to external storage, better data management
  • If corrupted: Attempt repair, but prepare for replacement

Prevention:

  • Regular backups (external drive or cloud)
  • Monitor drive space (don't let it get above 80% full)
  • Surge protector (prevents corruption from power issues)
  • Careful transport (physical shock damages HDDs)
  • Consider SSD for reliability (if replacing)

Pattern 4: Display Issues

Common symptoms:

  • Screen stays black (but computer powers on)
  • Flickering or intermittent display
  • Lines or artifacts on screen
  • Very dim display (backlight issue)
  • Colors wrong or washed out
  • Spots or clouding

Likely causes:

  • Loose display cable
  • Backlight failure
  • Display panel damage
  • Graphics hardware problem
  • Humidity damage

How to diagnose:

  • Test with external monitor (KEY TEST) - isolates display vs. graphics
    • If external works: problem is in display assembly
    • If external also fails: problem is in graphics hardware
  • Check brightness settings (surprisingly common oversight)
  • Gently press around screen bezel (loose connection?)
  • Note if problem is intermittent or constant
  • Look for physical damage (cracks, pressure marks)

Common context:

  • Physical pressure on closed lid
  • Humidity causing condensation
  • Display cable damage from opening/closing repeatedly
  • Impact or drop

Solutions:

  • If cable loose: reseating (may need professional)
  • If backlight failed: replacement (expensive)
  • If panel cracked: replacement
  • If graphics issue: may need motherboard repair/replacement
  • Workaround: External monitor while arranging repair

Prevention:

  • Protect laptop during transport
  • Don't put pressure on closed lid
  • Store in dry location (humidity control)
  • Open/close gently (cable stress)

Pattern 5: RAM Issues

Common symptoms:

  • Beep codes on startup (pattern varies by manufacturer)
  • Computer powers on but no display
  • Blue screen errors
  • Random crashes or freezes
  • Program crashes
  • Often happens after physical shock

Likely causes:

  • RAM not seated properly (became dislodged)
  • RAM module failure
  • RAM slot corrosion (humidity)
  • Incompatible RAM (if recently upgraded)

How to diagnose:

  • Listen for beep codes (usually indicate RAM)
  • Test by reseating RAM modules
  • Try one RAM stick at a time (if multiple)
  • Run memory diagnostic tool
  • Note if happened after transport or in humid conditions

Common context:

  • Rough transport (vibration unseats RAM)
  • Humid environments (corrosion)
  • After opening computer for other reasons

Solutions:

  • Reseat RAM (remove and reinstall firmly)
  • Clean contacts gently if corrosion visible
  • Replace RAM if module has failed
  • Ensure proper seating (should click into place)

Prevention:

  • Careful transport (padded case)
  • Humidity control
  • Ensure RAM properly seated after any internal work

Pattern 6: Power and Charging Problems

Common symptoms:

  • Computer won't turn on at all
  • Power light flickers or doesn't stay on
  • Computer dies immediately when unplugged
  • Charging light doesn't come on
  • Works only when plugged in
  • Burning smell from adapter

Likely causes:

  • Dead power adapter
  • Damaged charging port
  • Charging circuitry failure (motherboard)
  • Completely dead battery
  • Power button failure (rare)

How to diagnose:

  • Check adapter: Is it warm? Any damage to cable?
  • Check charging light: Does it come on when plugged in?
  • Try different outlet (eliminates wall power)
  • Wiggle connection (loose port?)
  • Smell adapter (burning smell = definitely bad)
  • Check if any response when power button pressed (lights? sounds?)

Common context:

  • Voltage fluctuations/power surges
  • Physical damage to adapter or port
  • Old adapter (cables fray over time)

Solutions:

  • Replace adapter (most common)
  • Port repair (may need professional)
  • Battery replacement
  • Motherboard repair (expensive, often not worth it)

Prevention:

  • Surge protector/voltage regulator (ESSENTIAL)
  • Unplug during storms
  • Cable management (don't stress connections)
  • Replace adapters at first sign of damage

Section 2.4: Remote Troubleshooting Skills

Time: 10 minutes

The Challenge of Remote Support

As a language technology consultant, you won't be physically present with the user. This means you need to:

  • Gather information through questions
  • Guide users through steps they can perform
  • Assess their comfort level with different tasks
  • Know when to escalate rather than push beyond user capability

Communication Strategies

1. Ask Clear, Specific Questions

Vague: "Is it working okay?"

Specific: "When you press the power button, what exactly happens? Do any lights come on? Do you hear any sounds?"

Vague: "Is it slow?"

Specific: "How long does it take from when you press the power button until you can start working? And once you're working, do programs respond quickly or is there delay?"

Vague: "Does it get hot?"

Specific: "Where exactly does it feel hot - bottom, sides, near the vents? Can you hold your hand there comfortably or is it too hot to touch?"

2. Give Step-by-Step Instructions

When guiding hardware procedures:

  • One step at a time
  • Very specific (not "check the RAM" but "look for small green circuit boards with chips on them")
  • Check understanding ("Do you see the component I described?")
  • Pause points ("Before you remove anything, take a photo so you remember how it goes back")
  • Safety warnings ("Make sure the computer is powered off and unplugged")

3. Assess Comfort Level

Before asking someone to open their computer:

  • "Have you ever opened the back of a laptop before?"
  • "How comfortable would you feel removing a couple of screws and looking inside?"
  • "I can walk you through this, but it's also okay to find someone local who can help if you're not comfortable with it."

If they're uncertain, provide alternatives:

  • "Let's try these external tests first"
  • "Is there anyone nearby who has done this before who could help?"
  • "This might be one to escalate to IT support"

4. Use Visual Aids

When possible:

  • Send photos or diagrams
  • Reference iFixit teardowns for their specific model
  • Video calls if bandwidth allows (watch what they're doing)
  • Ask them to take photos and send them

5. Document Everything

Good documentation helps:

  • Track patterns across users
  • Smooth escalation to IT if needed
  • Build organizational knowledge
  • Prevent repeat questions

Document:

  • Symptoms (specific details)
  • Tests performed
  • Results
  • Actions taken
  • Recommendations

Activity 2.4: Write Remote Instructions

Time: 5 minutes

Write step-by-step instructions for guiding a user through this task:

Task: Check their computer's vents for dust buildup and clean them externally.

Assume the user has never done this before. Write 5-7 steps that are clear enough for someone to follow via email or chat.

Sample Answer:

  1. Power off your computer completely and unplug the power adapter. Let it cool for at least 10 minutes.

  2. Find the ventilation slots on your computer - these are usually along the sides or back edge and look like narrow slits or grilles. Take a photo and send it to me if you're not sure you've found the right thing.

  3. Look at the vents in good light. Do you see dust or debris inside? Is it blocking the openings? (It's okay if there's a little dust, but we're looking for obvious buildup)

  4. If you see dust: Using a soft brush (like a clean paintbrush) or a dry cloth, gently brush away any dust from the outside. Brush AWAY from the openings so you're pulling dust out, not pushing it in.

  5. If you have compressed air: Hold the can upright, keep it about 6 inches from the vents, and use short bursts. Don't spray continuously or hold the can upside down. This helps blow dust out from inside.

  6. After cleaning, take another photo so I can see the result.

  7. Let me know if you noticed a lot of dust buildup - that might explain your overheating issues!

Key elements: Safety (power off, cool down), specificity (where vents are, what they look like), confirmation (photos), clear technique, connection to the problem.

Teaching Scenario 1: The Black Screen

Situation:

James in Papua New Guinea says his computer "won't turn on." When you ask him to describe exactly what happens, he says: "I press the power button and I can hear the fan running, and there's a small light on the front that comes on, but the screen stays completely black."

Context:

James works in a coastal area with high humidity. This morning he noticed some condensation on his desk when he came into the office.

Your turn: Work through the diagnostic process. Write down your answers before looking at the teaching points below.

  1. OBSERVE: What additional questions would you ask?
  2. ISOLATE: Based on symptoms, what components are most likely involved? What can you rule out?
  3. TEST: What tests would you recommend? (Put them in order)
  4. DECIDE: What would you recommend based on likely causes?

Teaching Points for Scenario 1:

OBSERVE questions:

  • Has this happened before?
  • Did anything unusual happen yesterday or last night?
  • Is there visible moisture on or around the computer?
  • Can you hear the hard drive spinning or any beep codes?
  • If you shine a flashlight on the screen, do you see anything faint?

ISOLATE:

Computer powers on (fan runs, power light on) which means power system works.

Screen stays black which means display or graphics or connection issue.

Given humidity context, likely moisture interfering with connections (display cable, RAM, or both).

Most likely: Loose or moisture-affected display connection or RAM

Can rule out: Power system (it's working), complete motherboard failure (fan runs)

Which pattern: This matches display issues (Pattern 4) or RAM issues (Pattern 5), complicated by environmental factors (humidity)

TEST (in order of least to most invasive):

  1. Let it dry: Power off completely, remove battery if possible, let sit for several hours in dry location
  2. External monitor test (if available): Would definitively separate display from graphics issue
  3. Listen for beeps: Any beep codes when starting?
  4. Flashlight test: Shine light on screen - if you can faintly see image, backlight failed
  5. If comfortable with opening: Reseat RAM (humidity can cause poor connection)

DECIDE:

Immediate:

  • Don't keep trying to start it (could cause damage if moisture present)
  • Let computer dry completely (several hours, warm dry location)
  • Try restarting after it's dry

If problem persists after drying:

  • If external monitor works, then display assembly issue (cable or panel)
  • If external monitor also doesn't work, then try reseating RAM
  • If RAM reseat doesn't help, then likely need IT support

Prevention:

  • Store computer in sealed container with silica gel
  • Allow computer to acclimate to room temp before starting
  • Work in driest location possible
  • Don't work immediately after moving from cold/humid environment

Key insight: Environmental context (humidity + condensation) immediately suggests moisture problem. Solution isn't technical repair, it's letting it dry! This is where understanding environmental factors really pays off.

Teaching Scenario 2: The Intermittent Shutdown

Situation:

David in Chad contacts you frustrated because his notebook has shut down unexpectedly three times this week while he was working. He says there's no warning—it just turns off suddenly. When he restarts it, sometimes it works fine for hours, other times it shuts down again within 30 minutes. He mentions that it seems to happen more often when he's running his translation software and has multiple documents open.

Context:

It's dry season, and dust is a constant problem. David works in a small office with no air conditioning. He's been in location for 8 months and hasn't done any maintenance on the computer.

Your turn: Apply the diagnostic framework.

  1. OBSERVE: What questions would you ask?
  2. ISOLATE: What's your primary hypothesis? Alternative possibilities?
  3. TEST: Design tests to confirm/rule out your hypothesis
  4. DECIDE: Recommended solution and if problem persists

Teaching Points for Scenario 2:

OBSERVE questions:

  • Is the fan very loud before shutdown?
  • Does the bottom/sides of the computer feel very hot?
  • Does it happen more in the afternoon (hottest time)?
  • Where do you work - on a desk, on your lap, on a cushion?
  • Are the vents blocked by anything?
  • Can you see dust in the vents?

ISOLATE:

Primary hypothesis: Overheating due to dust accumulation

  • Supports: Dry season, dust problem, 8 months no maintenance, shutdowns during intensive work, random timing
  • Pattern matches: Overheating (Pattern 2)

Alternative possibilities:

  • Power adapter failure (but would expect more consistent behavior)
  • RAM issue (but would expect different symptoms - crashes, not clean shutdowns)
  • Battery problem (but seems to happen on both battery and plugged in based on description)

Why overheating is most likely:

  • Thermal shutdowns are protective (computer shuts down before damage)
  • Worse with intensive work (generates more heat)
  • Random timing fits overheating (depends on ambient temp + workload)
  • Environmental context (dust, heat, no AC, no maintenance)

TEST (confirm hypothesis):

  1. Temperature check: Download and run temperature monitoring software (HWMonitor, Core Temp) - what temperatures right before shutdown? (Over 90°C confirms)

  2. Fan check: Is fan running loudly? Is it running at all?

  3. Visual inspection: Look at vents - is dust visible? Are they blocked?

  4. Pattern test: Does shutdown happen more during hot part of day? With intensive tasks? After longer work sessions?

  5. Position test: Does it happen more when laptop is on soft surface vs. hard desk?

DECIDE:

Immediate solution:

  1. Clean cooling vents:

    • External cleaning with brush/compressed air
    • Walk David through this step-by-step

  2. Improve airflow:

    • Use on hard, flat surface (not lap, not cushion)
    • Elevate slightly if possible (allows air under laptop)
    • Position away from walls (don't block vents)

  3. Reduce heat generation:

    • Close unnecessary programs
    • Work in coolest part of office
    • Take breaks (let computer cool)

If comfortable opening:

  1. Internal cleaning: Open bottom panel, clean dust from fan and heat sink

If problem persists after cleaning:

  • Fan might be failing (mechanical problem)
  • Thermal paste might need replacement (advanced)
  • Escalate to IT support

Prevention (critical for David):

  • Monthly: External vent cleaning
  • Quarterly: Internal cleaning (or more often in dusty season)
  • Daily habits: Work on proper surface, keep workspace clean, don't block vents, monitor fan noise
  • Environmental: Close windows during dust storms if possible

Key insight: This scenario teaches that simple maintenance (cleaning) solves many "hardware problems." Remote location + dusty environment + no maintenance = predictable overheating. The consultant's job includes helping users establish maintenance routines, not just fixing current problems.
