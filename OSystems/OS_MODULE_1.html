Section 1.2: What Does an Operating System Do?

Time: 15 minutes

Introduction

Before you can troubleshoot OS problems or configure systems effectively, you need to understand what an operating system actually does. Most people use computers daily without thinking about the OS - it's invisible until something goes wrong. In this section, you'll develop a clear mental model of the OS's role.

The OS as Bridge and Manager

Think of your operating system as the bridge between your computer's hardware and the applications you use. Without an OS, your hardware is just expensive metal and silicon. Without an OS, applications have no way to access the keyboard, display text on screen, save files, or connect to the internet.

The operating system manages three critical jobs:

1. Managing Hardware Resources

  • Controls access to the processor (CPU)
  • Allocates memory (RAM) to running programs
  • Manages storage (reading and writing files to disk)
  • Handles input devices (keyboard, mouse, touchscreen)
  • Controls output devices (display, speakers, printers)
  • Manages network connections

When you click to open a document, the OS finds the file on disk, loads it into memory, tells the processor to run the application, and displays the result on screen. The application doesn't talk directly to the hardware - the OS handles all of that.

2. Running and Managing Applications

  • Launches programs when you click them
  • Keeps multiple programs running simultaneously
  • Prevents programs from interfering with each other
  • Closes programs properly when you're done
  • Handles what happens when programs crash or freeze

When you have a word processor, email client, and web browser all open at once, the OS is juggling their competing demands for processor time, memory, and hardware access.

3. Providing the User Interface

  • Creates the desktop, windows, menus, and icons you interact with
  • Handles file management (folders, copying, deleting)
  • Manages user accounts and permissions
  • Provides system settings and control panels
  • Runs background services (like checking for updates or syncing files)

The "look and feel" of your computer - whether it's Windows, macOS, or Linux - is largely determined by how the OS presents itself to you.

Why This Matters for Language Technology Work

Understanding these three jobs helps you diagnose problems more effectively:

Hardware management issues show up as:

  • Computers that won't recognize USB drives or keyboards
  • Display problems (wrong resolution, can't detect external monitor)
  • Network connectivity failures
  • Printers that won't print

Application management issues show up as:

  • Programs that won't launch or crash repeatedly
  • "Not enough memory" errors
  • Slow performance when many programs are running
  • Programs that conflict with each other

User interface issues show up as:

  • Can't find files or folders
  • Settings that won't save
  • Language or keyboard input problems
  • Font display issues (critical for language work!)

When a language worker contacts you saying "my computer isn't working," your OS knowledge helps you ask the right questions to narrow down which of these three areas is causing the problem.

Activity 1.2a: Explore What's Running Right Now

Time: 8 minutes

Let's make this concrete by looking at what your operating system is doing right now.

Instructions:

For Windows users:
  1. Press Ctrl + Shift + Esc to open Task Manager
  2. Click "More details" if you see that option
  3. Look at the "Processes" tab
For macOS users:
  1. Press Command + Space and type "Activity Monitor"
  2. Press Enter to open it
  3. Look at the list of processes
For Linux users:
  1. Open your system monitor application (varies by distribution)
    • Ubuntu: Search for "System Monitor"
    • Other distros: May be called "Task Manager" or similar
  2. Look at the processes list
What you're seeing:

You're looking at all the programs and background services your OS is currently managing. Notice:

  • How many processes are running (probably dozens, even if you only opened a few programs)
  • Which ones are using CPU (processor) resources
  • Which ones are using memory (RAM)
  • Some you recognize (like your web browser) and many you don't (background system services)
In your learning journal, write:
  1. How many total processes are running on your system right now?
  2. What is using the most memory?
  3. What is using the most CPU?
  4. List 3-4 processes you don't recognize - these are likely OS services working in the background

Reflection: This is your OS at work - managing dozens of things simultaneously so you can focus on your actual work. When someone says their computer is "slow," you now have a window into what might be happening behind the scenes.

Here's a scenario to help you think about the OS's role:

Scenario: Maria is working on a translation project. She has her translation software open, a web browser with research materials, and a PDF document she's referencing. Suddenly her translation software freezes and won't respond.

Key Takeaways

  • The OS is the essential bridge between hardware and applications
  • It manages hardware resources, runs applications, and provides the user interface
  • Understanding these roles helps you diagnose where problems originate

  • When troubleshooting, ask yourself: Is this a hardware issue, an application issue, or a user interface issue?

Section 1.3: The Current OS Environment

Time: 20 minutes

Introduction

Now that you understand what an operating system does, let's look at the operating systems you'll actually encounter in your work as a language technology consultant. Understanding the strengths, limitations, and typical use cases of each OS helps you make informed recommendations and provide better support.

The Three Major Operating System Families

Language workers around the world use three main operating system families: Windows, macOS, and Linux. Each has different characteristics that make it more or less suitable for different contexts.

Windows

Current Versions:

  • Windows 11 (released 2021, current version)
  • Windows 10 (released 2015, still widely used, support until October 2025)
  • Older versions (Windows 7, 8, 8.1) are end-of-life and no longer receive security updates

Market Position:

Windows remains the dominant operating system globally, especially in Africa and the Americas where most language technology work happens. Many organizations standardize on Windows because:

  • It's pre-installed on most new computers
  • Most language software is developed for Windows first
  • IT departments are familiar with managing Windows networks
  • Hardware compatibility is broad

Strengths for Language Technology Work:

  • Wide software compatibility - most language tools run on Windows
  • Extensive font support and rendering capabilities
  • Good keyboard layout customization
  • Familiar to most users
  • Strong support for assistive technologies

Considerations:

  • Licensing costs (though many organizations have volume licenses)
  • Higher hardware requirements, especially for Windows 11
  • Regular updates can sometimes cause compatibility issues
  • Requires antivirus and security software

When Windows Makes Sense:

  • Organizations with existing Windows infrastructure
  • Users who need specific Windows-only language software
  • Contexts where IT support is Windows-focused
  • When budget allows for licensing and capable hardware

macOS

Current Versions:

  • macOS Sonoma (released 2023)
  • macOS Ventura (released 2022)
  • macOS Monterey (released 2021)
  • Apple typically supports the current version plus two previous versions

Market Position:

macOS has grown in popularity, especially among younger language workers and in contexts where font and typography work is central. Apple computers come with macOS pre-installed, and the OS is tightly integrated with Apple hardware.

Strengths for Language Technology Work:

  • Excellent font rendering and typography support
  • Superior handling of complex scripts and right-to-left languages
  • Built-in support for many languages and keyboards
  • Unix-based system (similar to Linux underneath)
  • Generally fewer security concerns than Windows
  • Long-term hardware reliability

Considerations:

  • Only runs on Apple hardware (significantly higher upfront cost)
  • Smaller selection of language-specific software
  • Some popular language tools are Windows-only
  • Repair and replacement can be expensive in remote locations
  • Less familiar to IT staff in some regions

When macOS Makes Sense:

  • Work heavily involves complex scripts, fonts, or typography
  • Budget allows for Apple hardware investment
  • User prefers the macOS interface and ecosystem
  • Language software needed is macOS-compatible
  • Long-term reliability is prioritized over initial cost

Linux

Common Distributions:

  • Ubuntu (most popular, user-friendly)
  • Linux Mint (similar to Windows interface)
  • Debian (stable, widely supported)
  • Fedora (cutting-edge features)
  • Many others, each with different characteristics

Market Position:

Linux has traditionally been less common for general language work, but it's growing in use for several important reasons:

  • Extends life of older hardware that can't run current Windows
  • No licensing costs
  • Increasing software compatibility
  • Strong in certain regions and technical communities

Strengths for Language Technology Work:

  • Free and open source (no licensing costs)
  • Runs well on older, less powerful hardware
  • Highly customizable
  • Strong security
  • Good font support (especially in recent versions)
  • Active community support
  • Can dual-boot with Windows on same computer

Considerations:

  • Steeper learning curve for users coming from Windows
  • Some language software doesn't run natively on Linux
  • May require workarounds (Wine, virtual machines) for Windows software
  • Less familiar to many IT support staff
  • Troubleshooting often requires command-line knowledge

When Linux Makes Sense:

  • Extending life of older computers
  • Budget constraints prevent Windows licensing or Mac hardware
  • Users are technically capable or have Linux-savvy support
  • Language software needs are met by Linux-compatible tools
  • Organization has commitment to open source software

Making OS Recommendations

As a language technology consultant, you'll sometimes be asked "What operating system should I use?" or "Should we upgrade?" Here are key questions to ask:

About the User:

  • What is their current OS and comfort level?
  • What language software do they need to run?
  • How technically confident are they?
  • Do they have local IT support? What OS does that support know?

About the Context:

  • What hardware is available or affordable?
  • Are there organizational standards or restrictions?
  • What do other team members use (for collaboration)?
  • Is internet connectivity reliable (for updates, cloud features)?

About the Work:

  • What scripts and languages will they work with?
  • Are they doing heavy font work or typography?
  • Do they need specific software that's OS-dependent?
  • How critical is system stability vs. having latest features?

There's rarely one "right" answer - it depends on weighing these factors for each specific situation.

Activity 1.3a: Identify Your Operating System

Time: 7 minutes

Let's get hands-on. You're going to identify exactly what operating system version you're currently running.

For Windows:
  1. Click the Start button (Windows icon)
  2. Type "About" and select "About your PC" or "System Information"
  3. Look for "Windows specifications" section
For macOS:
  1. Click the Apple menu (top-left corner)
  2. Select "About This Mac"
  3. You'll see the macOS version name and number
For Linux:
  1. Open a terminal window
  2. Type: lsb_release -a and press Enter
    • OR type: cat /etc/os-release and press Enter
  3. You'll see your distribution name and version
In your learning journal, document:
  1. Operating System: (e.g., Windows 11, macOS Sonoma, Ubuntu 22.04)
  2. Version Number: (e.g., 23H2, 14.1, 22.04.3)
  3. Architecture: 64-bit or 32-bit (usually shown in same location)
  4. Build Number: (if shown - this is the specific build of your OS version)

Why this matters: When troubleshooting problems or checking software compatibility, you need to know exactly what OS version you're running. "I have Windows" isn't specific enough - Windows 10 and Windows 11 have different capabilities and requirements.

There's no single "right" answer here - this is about thinking through the factors that matter in real-world situations.

Key Takeaways

  • Windows, macOS, and Linux each have distinct strengths and appropriate use cases
  • Windows remains dominant but macOS and Linux are viable alternatives in specific contexts
  • OS recommendations depend on user needs, hardware, budget, software requirements, and available support
  • Knowing exactly what OS version someone is running is essential for troubleshooting and compatibility checking
  • The "best" OS is the one that fits the specific context and constraints

Section 1.4: System Information and Compatibility

Time: 15 minutes

Introduction

Identifying your OS version (as you did in the previous section) is just the beginning. As a language technology consultant, you need to know how to gather detailed system information and assess whether a system meets the requirements for specific language software. This section teaches you where to find that information and how to use it.

Why System Information Matters

When someone contacts you with a problem or asks if they can run certain software, you need more than just "I have Windows 10." You need to know:

  • Processor type and speed (affects performance)
  • Amount of RAM (determines what can run simultaneously)
  • Available storage space (for installing software and storing files)
  • Graphics capabilities (especially for display-intensive work)
  • Current updates installed (some problems are fixed by updates)

You also need this information to:

  • Determine if their system meets software requirements
  • Diagnose performance problems
  • Decide if an upgrade is feasible
  • Plan for new software installations
  • Help users understand their system's capabilities and limitations

Finding Detailed System Information

Each operating system provides tools to view comprehensive system information. Let's look at where to find them.

Windows: System Information Tool

Windows has a built-in System Information utility that shows detailed specifications:

How to access:
  1. Press Windows key + R to open Run dialog
  2. Type msinfo32 and press Enter
  3. The System Information window opens
What you'll see:
  • System Summary (first screen):
    • OS Name and Version
    • System Manufacturer and Model
    • Processor (CPU) details
    • Installed Physical Memory (RAM)
    • BIOS Version/Date
  • Components section (expandable menu on left):
    • Storage devices
    • Display adapters
    • Network adapters
    • USB devices
  • Software Environment section:
    • Running tasks
    • Startup programs
    • System drivers

Quick alternative: Right-click on "This PC" or "My Computer" → Properties shows basic information.

macOS: System Information

macOS provides system information through "About This Mac" and a more detailed System Information utility.

Basic information:
  1. Click Apple menu → About This Mac
  2. You'll see: macOS version, processor, memory, graphics
Detailed information:
  1. From "About This Mac," click "System Report" button
  2. OR: Hold Option key, click Apple menu, select "System Information"
  3. This opens detailed system profiler
What you'll see:
  • Hardware section:
    • Model identifier
    • Processor details
    • Memory specifications
    • Storage devices
    • Graphics cards
  • Network section:
    • Wi-Fi information
    • Network locations
  • Software section:
    • System version
    • Applications installed
    • Extensions and frameworks

Linux: System Information Commands and Tools

Linux provides system information through command-line tools and graphical applications.

Command-line methods:

For basic system info:

uname -a           # Kernel and system information
lscpu              # CPU information
free -h            # Memory information
df -h              # Disk space information
lsb_release -a     # Distribution information

For hardware details:

sudo lshw          # Comprehensive hardware list
lspci              # PCI devices (graphics, network, etc.)
lsusb              # USB devices
Graphical tools (varies by distribution):
  • Ubuntu: "Settings" → "About" shows basic info
  • System Monitor shows real-time resource usage
  • HardInfo (install separately) provides detailed hardware information

Understanding System Requirements

Software developers specify minimum and recommended requirements for their applications. Here's how to read and assess them:

Typical requirement specifications look like:
Minimum Requirements:
- Operating System: Windows 10 (64-bit) or later
- Processor: Intel Core i3 or equivalent
- Memory: 4 GB RAM
- Storage: 2 GB available space
- Display: 1280x720 resolution

Recommended Requirements:
- Operating System: Windows 11 (64-bit)
- Processor: Intel Core i5 or equivalent
- Memory: 8 GB RAM
- Storage: 5 GB available space (SSD preferred)
- Display: 1920x1080 resolution
How to interpret this:
  • Minimum = Software will run, but may be slow or limited
  • Recommended = Software will run well with good performance
  • 64-bit = Most modern systems, but verify in your system information
  • Processor equivalents = "or equivalent" means comparable AMD or other processors
  • SSD preferred = Faster than traditional hard drives, but not required
Common compatibility issues:
1. 32-bit vs 64-bit:
  • Most new software requires 64-bit OS
  • 32-bit systems cannot run 64-bit software
  • Check your system architecture in system information
2. RAM limitations:
  • 4GB RAM is often minimum, but tight for multitasking
  • 8GB is comfortable for most language work
  • More RAM helps with large documents, multiple programs
3. Storage space:
  • Check not just total space, but available space
  • Consider space for data files, not just the program
  • System drive (C: on Windows) needs breathing room
4. OS version specifics:
  • "Windows 10 or later" means Windows 11 is fine
  • "macOS 12 or later" means Ventura (13) and Sonoma (14) work
  • Linux requirements may specify kernel version or specific distributions

Activity 1.4a: Document Your System Specifications

Time: 8 minutes

Now you're going to gather and document detailed information about your own system. This creates a reference you can use anytime you need to check compatibility or troubleshoot.

Instructions:

Use the appropriate system information tool for your OS (described above), and document the following in your learning journal:

Basic Specifications:
  1. Operating System: (full name and version)
  2. Processor (CPU):
    • Brand and model (e.g., Intel Core i5-8250U)
    • Speed (e.g., 1.6 GHz)
    • Number of cores (e.g., 4 cores)
  3. Memory (RAM):
    • Total installed (e.g., 8 GB)

    • Available free space (e.g., 128 GB free)
  5. Display:
    • Screen resolution (e.g., 1920 x 1080)
    • Graphics/video card if shown
Additional Information:
  1. System manufacturer and model (e.g., Dell Latitude 5400)
  2. 64-bit or 32-bit architecture

Tip: Take a screenshot of your system information window and save it with your learning journal. This gives you a quick visual reference.

Activity 1.4b: Assess Compatibility

Time: 5 minutes

Now let's practice using system information to assess software compatibility.

Scenario: A language worker wants to install Paratext 9 on their computer. Here are Paratext's actual requirements:

Paratext 9 Requirements:
  • Operating System: Windows 10 (64-bit) or later, OR macOS 10.13 or later
  • Processor: 1 GHz or faster
  • Memory: 2 GB RAM (4 GB recommended)
  • Storage: 1 GB available space
  • Display: 1024x768 resolution minimum
  • .NET Framework 4.8 or later (Windows) / Mono (Linux)
In your learning journal, answer:
  1. Based on your own system specifications (from Activity 1.4a), can your computer run Paratext 9?
    • Check each requirement against your specifications
    • Identify any that don't meet minimum requirements
    • Identify any that don't meet recommended requirements
  2. If your system doesn't meet all requirements, what would need to change?
    • Which component is insufficient?
    • Would an upgrade be possible, or would a new computer be needed?
  3. For Pastor John's computer from Activity 1.3b (7-year-old Dell, Windows 7, 4GB RAM, Core i3):
    • Which requirements does it meet?
    • Which requirements does it fail?
    • What's the main barrier to running Paratext 9?

Think about: Sometimes a system technically meets minimum requirements but wouldn't provide a good user experience. How would you communicate that to a user?

Key Takeaways

  • Every operating system has tools to view detailed system specifications
  • Key specifications include: OS version, processor, RAM, storage, and display
  • Software requirements specify minimum (barely runs) and recommended (runs well) specifications
  • 64-bit vs 32-bit architecture is a critical compatibility factor
  • Knowing your system specifications helps you assess compatibility before installing software
  • System information is essential for troubleshooting and making upgrade decisions
